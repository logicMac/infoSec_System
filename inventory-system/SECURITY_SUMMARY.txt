🔐 SECURITY FLOW
================================================================================

LOGIN FLOW:
1. User enters username/password
2. System validates credentials
3. Rate limit check (5 attempts / 5 min)
4. CSRF token validation
5. OTP generated and sent to email
6. User redirected to OTP verification page
7. User enters OTP (max 3 attempts, 5 min expiry)
8. OTP validated
9. Session created with security features
10. User redirected to role-based dashboard

PROTECTED PAGE ACCESS:
1. User requests protected page
2. auth_middleware.php checks session
3. Session validated (timeout, integrity)
4. Role checked against route requirements
5. Access granted or denied
6. Session ID regenerated periodically

API REQUEST:
1. Request received
2. Rate limit checked
3. Authentication validated
4. CSRF token checked (POST/PUT/DELETE)
5. Origin validated
6. Input sanitized
7. Request processed
8. Secure response sent with headers

🔑 KEY FUNCTIONS
================================================================================

AUTHENTICATION:

- requireRole($role)      - Check specific role
- requireAdmin()          - Admin only
- requireStaff()          - Staff or admin

CSRF:

- validateCSRFToken()     - Validate token
- csrfField()             - Output hidden field

RATE LIMITING:

OTP:

- sendOTPEmail()          - Send via email

SESSION:

- regenerateSession()     - Regenerate ID

API:

- sendJSONResponse()      - Send secure response

🆘 TROUBLESHOOTING
================================================================================

OTP NOT WORKING:
- Check logs/otp_log.txt for generated codes
- Verify email configuration
- Check PHP error logs

RATE LIMIT TOO STRICT:
- Clear browser cookies
- Adjust limits in inc/security.php
- Wait for cooldown period

SESSION EXPIRES QUICKLY:
- Increase timeout in inc/security.php
- Check server session settings

CSRF ERRORS:
- Ensure csrfField() in all forms
- Check session is started
- Clear browser cache
